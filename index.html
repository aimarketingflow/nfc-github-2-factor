Important: This system requires a physical NFC reader and blank NFC tags. The security model depends on keeping your NFC tag physically secure.

How It Works

  1. Generate an SSH key protected by your NFC tag's unique ID
  2. The NFC ID serves as the passphrase but is never displayed
  3. Unlock your key by scanning the same physical NFC tag
  4. Use for GitHub, GitLab, servers, or any SSH authentication

Security Architecture

üõ°Ô∏è Security Guarantees

Zero-Visibility

NFC value never displayed, logged, or saved

Physical Possession

Requires the actual NFC tag to authenticate

No Network Exposure

Authentication happens entirely locally

Replay Protection

Direct physical scan required each time

‚ö†Ô∏è Security Considerations

Physical Security: Your NFC tag is like a physical key. Keep it secure and never let it out of your possession.
No Recovery: If you lose your NFC tag, you cannot recover the SSH key. Always maintain backup authentication methods.
Unique Tags: Use different NFC tags for different services to limit exposure if one is compromised.

‚úÖ Best Practices

  • Store NFC tags in a secure physical location
  • Never photograph or scan tags with smartphones
  • Use write-protected tags when possible
  • Rotate keys and tags every 90 days
  • Maintain offline backup authentication
  • Monitor SSH logs for unauthorized access attempts
