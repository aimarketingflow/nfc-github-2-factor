Ultra-Secure SSH Authentication Using Physical NFC Tags

Executive Summary

The NFC GitHub 2FA system provides unprecedented security for SSH authentication by requiring physical possession of an NFC tag, eliminating remote attack vectors while maintaining complete invisibility of credentials.

Security Architecture

Protected AgainstSecurity Measure
Remote AttacksPhysical tag required
KeyloggersNo keyboard input
Screen CaptureNFC value never displayed
Credential TheftPassphrase on physical tag only

System Requirements

  • NFC Reader (ACR122U or compatible)
  • Blank NFC Tags (NTAG213/215/216)
  • Python 3.x, OpenSSH, Git
  • Linux, macOS, or Raspberry Pi

Installation Guide

1Linux Installation

$ sudo apt-get update $ sudo apt-get install -y python3 git openssh-client $ sudo apt-get install -y libnfc-bin libnfc-dev pcscd pcsc-tools $ sudo systemctl enable --now pcscd

2macOS Installation

$ brew install python3 git libnfc # Grant Terminal.app Input Monitoring permission in System Preferences

3Clone Repository

$ git clone https://github.com/aimarketingflow/nfc-github-2-factor.git $ cd nfc-github-2-factor && chmod +x scripts/*.sh

4Verify NFC Reader

$ nfc-list NFC device: ACS / ACR122U PICC Interface opened
